# 말투 프로필 증분 갱신 (Slack → 위키)

어제 하루 내 Slack 메시지에서 어투 신호를 관찰해, 위키 말투 프로필을 **증분** 갱신한다.
대상 문서: `~/ryu-mg/wiki/concepts/bao-slack-writing-style.md` (repo `ryu-mg/llm-wiki`).

**핵심 원칙 — 유지가 기본, 변화만 반영:** 이 스킬은 매일 돌지만 문서를 매번 다시 쓰지 않는다.
하루치 노이즈(일회성 농담·특이 표현)로 규칙을 흔들지 말고, **여러 날 반복 관측되는 뚜렷한 변화**만 규칙에 반영한다. 대부분의 날은 "변화 없음, 유지"가 정답이다.

## 0. 항상 먼저

위키 스키마 `~/ryu-mg/wiki/CLAUDE.md`를 읽고 frontmatter·언어(한국어 기본, 기술용어 원문)·`log.md` append 규칙을 따른다.

## 1. 날짜 결정

- 인자로 날짜(`YYYY-MM-DD` 또는 "어제"/"6월 28일")를 받으면 그 날짜.
- 없으면 **어제**: `date -v-1d +%F` (macOS).

## 2. 신원 확인

- 내 Slack `user_id`는 `slack_search_public_and_private` 도구 설명의 *"Current logged in user's user_id is ..."* 에서 매번 읽는다. 하드코딩 금지.

## 3. Slack 조회

`slack_search_public_and_private`:

```
query: from:<@USER_ID> on:YYYY-MM-DD
sort: timestamp, sort_dir: asc, limit: 20, include_context: false
```

- 페이지가 더 있으면 `cursor`로 이어 받아 그날 메시지를 모두 수집.
- **반드시 제외**: 봇 자동 메시지(근태 `#나는-일하고-있다`의 "자리 비워요/복귀했어요", daily-scrum 자동 포스트 등 `[BOT]` 표시분). 이건 내 어투가 아니다.
- **활용**: 사람이 직접 친 채널/스레드/DM 메시지의 어투만 관찰. PR 알림 같은 스킬 자동 산출물도 어투 판단에서 제외(형식이 고정돼 있어 말투 신호가 아님).

## 4. 어투 신호 관찰

기존 문서의 규칙 축(종결어미, 문장부호 습관, "요-" 지시어, 감탄사·초성, 이모지, 요청 톤, 이중 레지스터)을 기준으로 어제 메시지를 대조한다:

- 새로 자주 나온 표현/어미/이모지가 있는가?
- 기존 규칙과 어긋나는 패턴이 반복되는가? (1~2번은 노이즈, 3번 이상 또는 최근 여러 날 연속이면 신호)
- 빈도 변화(예: "~용" 늘리기가 줄고 "~요"가 늘었다)가 뚜렷한가?

## 5. 갱신 판단 (보수적으로)

1. **기존 문서를 읽는다** (`concepts/bao-slack-writing-style.md`).
2. 관측을 3분류:
   - **유지**: 기존 규칙으로 설명됨 → 아무것도 바꾸지 않는다.
   - **보강**: 기존 규칙의 예시가 부족한데 좋은 새 실제 예시가 있음 → 예시만 1개 추가(규칙 골격 유지).
   - **변경**: 여러 날 반복되는 뚜렷한 어투 변화 → 해당 규칙을 수정. **한 번에 규칙 골격을 갈아엎지 않는다.**
3. 변경/보강이 있을 때만 문서 본문 수정 + frontmatter `updated`를 오늘로.
4. **변화 로그**: 문서에 `## 변화 로그` 섹션이 없으면 만들고, 갱신한 날만 한 줄 append:
   ```
   - 2026-07-03: (변경 요지) 예) 어미 "~용" 빈도↓, "~요" 기본화 경향 — §1 갱신
   ```
   유지로 끝난 날은 로그도 남기지 않는다(노이즈 방지).

## 6. 색인 + 커밋

- `index.md`의 이 항목 요약은 규칙 골격이 바뀐 경우만 갱신. 사소한 예시 추가면 건드리지 않는다.
- 변경이 있었으면 `log.md` 맨 아래 append:
  ```
  ## [<오늘 날짜>] update | 말투 프로필 갱신 (YYYY-MM-DD 관측)
  - 수정: [[concepts/bao-slack-writing-style]] — (변경 요지 1줄)
  ```
- **변경이 없으면 커밋하지 않는다.** 보고만: "YYYY-MM-DD 관측 — 변화 없음, 유지".
- 변경이 있으면:
  ```bash
  cd ~/ryu-mg/wiki && git add -A && git commit -m "update: 말투 프로필 갱신 (YYYY-MM-DD 관측)" && git push
  ```

## 금기

- 하루치 노이즈로 규칙 골격 변경 금지. 매일 문서 통째 재작성 금지. 유지가 정답인 날이 대부분.
- 봇 자동 메시지·스킬 산출물을 어투로 오인 금지.
- 확실하지 않으면 유지 + "확인 필요"로 보고. 억지 변경 금지.

## 루틴 등록 (참고)

매일 자동 실행은 `/schedule`(cron)로 이 스킬을 날짜 인자 없이(=어제) 돌게 건다. 등록은 이 스킬 밖의 일.
